Dept. 06 // Engineering — Facility documentation archive

What makes this different

SLOP maintains detailed architectural records for all company facilities. These records are mostly accurate and somewhat current. Engineering personnel may access facility blueprints for restoration purposes. Some records have experienced convenient data corruption and may not reflect current structural conditions.

The short version

Every reclaimable building in Slopworks is modeled from real architectural data. Real Revit and Navisworks building models — the same files architects and engineers use to design actual buildings — are converted into explorable game levels. Real duct layouts, real pipe routing, real mechanical rooms, real electrical systems. No other game does this.

Environmental authenticity

Why real buildings matter

Exploration

Every building feels different

Because they ARE different real buildings. A hospital layout is nothing like a warehouse. A power plant has spaces a machine shop doesn't. The variety is architectural, not algorithmic.

Restoration

Real systems to fix

MEP restoration is grounded in actual building systems. The duct you're repairing follows a real routing path. The pipe manifold has valves where an actual building would. The electrical panel serves actual zones.

Scale

Thousands of source buildings

The BIM data library includes thousands of real building models — schools, hospitals, factories, offices, warehouses, power plants. Each one is a potential game level with unique architecture.

Atmosphere

Authentic industrial spaces

A mechanical room designed by a real engineer has a complexity and logic that procedural generation can't match. Pipes go somewhere. Ducts serve a purpose. The space makes sense even in ruin.

Technical pipeline

How real becomes game

The BIM-to-game pipeline converts architectural data into explorable, gameplay-ready levels.

→

Revit / Navisworks model

Start with a real building model created by architects and engineers. Complete with structural, mechanical, electrical, and plumbing systems.

→

Export and import

Convert to a Unity-compatible format. Mesh cleanup, material assignment, LOD generation for performance.

→

System identification

Automated detection of MEP systems — which pipes are restorable, which ducts carry air, which panels distribute power. These become gameplay objectives.

→

Ruin generation

Procedural damage: remove mesh sections for collapse, add vegetation scatter, apply corrosion materials, flood low points. The clean building becomes a ruin.

→

Gameplay population

Fauna spawn placement near mechanical rooms and dark areas. Hazard zones. Loot distribution. NavMesh baking for creature AI. The building becomes a dungeon.

→

Narrative layer

AI-generated building dossier from the architectural metadata. Who built it, what it produced, what happened. Environmental storytelling through logs, posters, and artifacts.
