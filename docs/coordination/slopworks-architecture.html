Hard rule

Never push directly to master

Both agents push to their branch, then gh pr create --base master. The MasterPlaytest scene validates that all features coexist before the PR can merge. Use gh pr merge after review.

Shared infrastructure

PlaytestBootstrap.cs

Plain C# class (not MonoBehaviour). Called once by whoever owns the scene. Creates: grid, simulation, player, HUD, registries, combat template, weapon. Returns PlaytestContext.

Shared tool handling

PlaytestToolController.cs

Added via AddComponent. Handles all structural/automation tools, ghost previews, belt visuals, shared OnGUI stats. Also provides static helpers: CreateSharedBuildPage(), CreateGroundPlane(), BakeNavMesh().

IPlaytestFeatureProvider & dual-mode pattern

public interface IPlaytestFeatureProvider
{
  string ProviderName { get; }

  // Setup phases (called by orchestrator in order)
  void CreateDefinitions(PlaytestContext ctx);
  void ConfigureBuildPage(HotbarPage buildPage);
  void RegisterToolHandlers(PlaytestToolController toolCtrl);
  void CreateWorldObjects(PlaytestContext ctx, PlaytestToolController toolCtrl);
  WaveControllerBehaviour CreateCombatSetup(PlaytestContext ctx);
  void PreSeed(PlaytestToolController toolCtrl);
  IEnumerator WireHUD(PlaytestContext ctx);

  // Runtime callbacks
  void FixedTick(float deltaTime);
  void UpdateInput(Keyboard kb);
  void DrawGUI(PlaytestToolController toolCtrl, ref float y, ...);
  void Cleanup();
}

KevinPlaytest scene

  • PlaytestBootstrap.Setup()
  • CreateGroundPlane()
  • Creates tool controller
  • Calls own interface methods internally
  • Owns SO cleanup in OnDestroy

FixedUpdate / Update / OnGUI active

Provider mode

MasterPlaytest scene

_isStandalone = false

Awake detects MasterPlaytestSetup:

  • Returns immediately from Awake
  • Orchestrator calls interface methods
  • Orchestrator handles bootstrap once
  • Orchestrator owns SO cleanup

FixedUpdate / Update / OnGUI suppressed

Key insight

The guard pattern

In Awake: if (GetComponent<MasterPlaytestSetup>() != null) { _isStandalone = false; return; }
This single check makes each bootstrapper work in both modes. No configuration needed -- component composition on the GameObject determines behavior.

MasterPlaytest root GameObject

One GameObject, five components. Order matters -- MasterPlaytestSetup runs Awake first, sets _isStandalone = false on both providers before their Awake fires.

MasterPlaytestSetup orchestrator -- calls providers in phase order
KevinPlaytestSetup provider -- buildings, supply chain, overworld
JoePlaytestSetup provider -- turrets (skeleton, Joe fills in)
PlaytestLogger toggleable verbose runtime logging
PlaytestValidator automated checks -- runs ValidateMasterScene
Orchestrator owns

MasterPlaytestSetup responsibilities

  • Calls PlaytestBootstrap.Setup() once
  • Creates ground plane and build page
  • Creates PlaytestToolController
  • Discovers providers via GetComponents<IPlaytestFeatureProvider>()
  • Calls each phase on all providers
  • Bakes NavMesh after all world objects
  • Delegates FixedUpdate/Update/OnGUI to providers
  • Owns shared SO cleanup in OnDestroy
Wave deconfliction

No duplicate wave controllers

Both bootstrappers create identical home-base wave controllers in standalone mode.

In master mode:

  • Kevin's CreateCombatSetup returns his wave controller (+ building enemies)
  • Joe's CreateCombatSetup checks _isStandalone -- returns null in master mode
  • Orchestrator takes first non-null result
  • Validator checks: at most 2 wave controllers (home-base + building)

7-phase lifecycle

MasterPlaytestSetup.Awake() orchestrates all providers through these phases in order. Each phase completes for ALL providers before the next begins.

CreateDefinitions
Create ScriptableObject definitions (building defs, turret defs). Runtime SOs that define game objects.
Kevin: warehouse, supply dock Joe: turrets (future)
ConfigureBuildPage
Add dev-specific entries to the shared build page (slots 7+). Shared slots 0-6 are already set by CreateSharedBuildPage().
Joe: slot 7 turret (future)
RegisterToolHandlers
Register custom tool input handlers on the PlaytestToolController. Also caches _toolCtrl reference.
Joe: turret placement (future)
CreateWorldObjects
Create physical objects in the scene: buildings, portals, world items, storage, supply chain. Runs AFTER tool controller exists.
Kevin: warehouse, MEP, portals, dock, supply chain
CreateCombatSetup
Create spawn points, wave controllers, building enemies. First non-null WaveControllerBehaviour wins. Joe returns null in master mode.
Kevin: home waves + building enemies Joe: null (master)
PreSeed
Optional factory pre-seeding (only when _preSeedFactory is true). Shared pre-seed runs first, then each provider adds on top.
Shared: PreSeedFactory() Joe: turret chain (future)
WireHUD
Wire dev-specific HUD extensions via coroutine. Orchestrator handles the 2-frame delay. Provider bodies run synchronously (yield break).
Kevin: building status callbacks Joe: turret HUD (future)
